## Communication

**Primary channel:** Teams Group "Turing Titans — Film Adviser"

**Standup cadence:**  
Monday, Wednesday, Friday at 21:00, quick sync (10–15 minutes).  
If unavailable, a written update must be posted before the meeting time.

**Response time expectation:**  
Messages should be acknowledged within **3 hours (10:00–23:00)**.  
Late-night messages (after 23:00) should be acknowledged by **10:00 next morning**.

**Meeting platform:**  
Primarily Discord or Google Meet. In-person meetings if all members are available locally.

**If someone is unresponsive:**  
- After 12 hours → send follow-up message  
- After 24 hours (with deadline near) → escalate within team  
- If still unresolved → notify instructor to document issue  

---

## Decision-Making

**Small decisions** (UI tweaks, minor backend changes, implementation details):  
→ Any two members can agree and proceed.

**Core product decisions** (features, AI behavior, UX flow):  
→ Discussed by all members, decided by majority.

**Critical technical decisions** (AI scoring logic, system architecture):  
→ Led by Giorgi (AI Lead), but must be explained and agreed upon by team.

**Major changes / pivots:**  
→ Require:
- Dedicated meeting (minimum 30 minutes)   
- Majority agreement  

We do not make major decisions via chat only.

---

## Conflict Resolution

We follow a structured approach:

**Step 1 — Direct Discussion (15 min):**  
Each person explains their perspective without interruption.

**Step 2 — Written Clarification:**  
Each party writes a short explanation of their reasoning.

**Step 3 — Cool-off (12–24 hours):**  
No immediate decisions; revisit after reflection.

**Step 4 — Final Decision:**  
If unresolved:
- Technical conflicts → Giorgi (AI/backend decisions)
- UI/UX conflicts → Gela
- Data/privacy conflicts → Ivane  

All team members agree to respect the final decision.

**Escalation:**  
If conflict cannot be resolved or affects progress → contact course instructor.

## Contribution and Accountability

**Work tracking:**  
- All work must be committed to GitHub  
- Each member commits their own work  
- No “ghost contributions”

**Repository rules:**  
- Clear commit messages required  
- Feature branches preferred for major changes  
- Code must be understandable by others  

**Deadlines:**  
- Work must be completed **at least 6 hours before submission**  
- Final system check **3 hours before deadline**

**Responsibility split:**
- Giorgi → AI + backend logic must be functional and stable  
- Gela → UI must be usable and connected to backend  
- Ivane → database and data flow must be correct and secure  

**Underperformance handling:**  
- First issue → internal discussion  
- Repeated issues → responsibilities reassigned  
- Final evaluation → contribution reflected honestly  

---

## Team Norms

- We prioritize **working product over perfect ideas**
- We do not overcomplicate AI — focus on reliability first
- We explain technical decisions clearly to each other
- We test features before saying they are “done”
- We communicate early if blocked — no silent delays
- We aim to finish work **before deadline day**, not at the last minute
- We give **specific, constructive feedback**, not vague criticism

---

## Project-Specific Agreements (Film Adviser)

- AI must always return **structured JSON** or trigger fallback  
- Fallback system must always produce usable recommendations  
- Privacy is critical — only necessary user data is stored  
- System must remain usable even if AI fails  
- Fairness scoring is a **core feature and priority**
